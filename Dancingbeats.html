Dancing Beats

- Saranya and Nalini

Dancing beats represents a music visualization along with the study of various attributes such as structures, beats etc. based on a music dataset. The dataset will reflect in the varied visualizations devised according to sound. Coupling hearing and vision through these graphical representations can help an individual percept dramatically by synchronizing them in a meaningful way. This will be implemented using a combination of d3 and HTML5 Web Audio API tools. Each visualization will use the dataset differently using a trial and error approach.

About

Stacked Bar Chart: The visualization of a few songs from the music dataset are pictorially represented. Each column represents group of SVG rectangle elements along the time axis. The dataset is generated in the form of columns representing different types of features of a song (Structure -> Silence, Instrumental, Chorus, Verse) and beats (Beats -> 1 to 4), showing about a total of eight path elements.

Line Graph: A multi-series line chart is a set of path elements representing the lines. We use the "cardinal" interpolator which gives a smoother more rounded appearance to the lines than using "linear" which would give straight line segments from point to point.

Streamgraph: The streamgraph is made up of different path elements for each of the series in the data. Instead of drawing lines, the paths create an irregular shape that fills the area taken up by the series. To draw the path, the d3.layout.stack() and d3.svg.area() tools are used. The data columns represented are the same as that of a bar chart.

Stacked Area Chart: There is only one difference when compared to that of a Streamgraph. Here only a single line is present. The type of offset that is sent to the "stack" is changed. The "zero" offset is used instead of a "wiggle". The data columns represented are same as that of a bar chart.


Multi-Line Chart: A multi-series line chart is a set of path elements representing the lines. We use the "cardinal" interpolator which gives smoother and more rounded appearance to the lines than using "linear" which would give straight line segments from point to point.

Steps for building the chart

The   basic steps for building the chart are as follows: (1) An Array is derived using variable names that represent the series in the data file (2) The data is then transformed into the the input D3 needs to layout the chart and (3) D3 is to create the SVG elements.
